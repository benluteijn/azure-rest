              CognitiveServicesManagementClient

              Create Cognitive Services Account. Accounts is a resource group wide resource type. It holds the keys for developer to access intelligent APIs. It's also the resource type for billing.

              Request

              PUT /subscriptions/{subscriptionId}/resourceGroups/{resourceGroupName}/providers/Microsoft.CognitiveServices/accounts/{accountName}?api-version

              Properties

                1. resourceGroupName (required)
                2. string
                The name of the resource group within the user's subscription.
                1. accountName (required)
                2. string
                The name of the cognitive services account within the specified resource group. Cognitive Services account names must be between 3 and 24 characters in length and use numbers and lower-case letters only.
                1. parameters (required)
                The parameters to provide for the created account.

                view child properties

                  1. sku (required)
                  The SKU of the cognitive services account.
                  1. kind (required)
                  2. string
                  Required. Indicates the type of cognitive service account.
                  1. location (required)
                  2. string
                  Required. Gets or sets the location of the resource. This will be one of the supported and registered Azure Geo Regions (e.g. West US, East US, Southeast Asia, etc.). The geo region of a resource cannot be changed once it is created, but if an identical geo region is specified on update the request will succeed.
                  1. tags
                  2. object
                  Gets or sets a list of key value pairs that describe the resource. These tags can be used in viewing and grouping this resource (across resource groups). A maximum of 15 tags can be provided for a resource. Each tag must have a key no greater than 128 characters and value no greater than 256 characters.
                  1. properties (required)
                  Must exist in the request. Must not be null.
                1. api-version (required)
                2. string
                Version of the API to be used with the client request. Current version is 2016-02-01-preview
                1. subscriptionId (required)
                2. string
                Azure Subscription ID.

              Returns

              HTTP RESPONSE CODES
                1. 200
                If resource is created successfully or already existed, the service should return 200 (OK).
                1. 201
                If resource is created successfully, the service should return 201 (OK).
                1. default
